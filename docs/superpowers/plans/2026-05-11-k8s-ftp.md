# k8s Plain FTP Service Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** k8s 클러스터 내부에 단일 Pod vsftpd 기반 Plain FTP 서비스를 구축한다. 가상 사용자별 chroot 격리, 무중단 사용자 추가, ACTIVE/PASV 모드, 500+ 동시 접속을 지원한다.

**Architecture:** Debian 12-slim 기반 자체 빌드 vsftpd 컨테이너 + NAS StorageClass PVC + user-syncer 사이드카(무중단 사용자 동기화) + 사내 LoadBalancer + Calico EgressGateway(ACTIVE 모드 source IP 매칭). 단일 Pod 수직 스케일로 500+ 동시 접속 처리.

**Tech Stack:** Debian 12-slim, vsftpd, pam_userdb (libpam-modules, Berkeley DB), inotify-tools, Kubernetes (Deployment, Secret, ConfigMap, PVC, LoadBalancer Service), Calico Enterprise EgressGateway.

**베이스 이미지 결정 노트 (2026-05-11)**: 최초 plan은 Alpine 3.20을 베이스로 명시했으나, Alpine의 `linux-pam` 패키지는 `pam_userdb.so`를 빌드 시 포함하지 않아 가상 사용자 인증을 그대로 구현할 수 없음. Debian 12-slim은 `libpam-modules` 패키지로 `pam_userdb.so`를 정상 제공. 이미지 크기 ~22MB → ~80MB로 증가하나 spec의 가상 사용자 + 무중단 동기화 모델을 그대로 유지한다.

**Reference:** 설계 문서 `docs/superpowers/specs/2026-05-07-k8s-ftp-design.md`

### Task 4: entrypoint.sh 작성

**Files:**
- Create: `docker/entrypoint.sh`

- [ ] **Step 1: entrypoint 작성**

Write `docker/entrypoint.sh`:
```bash
#!/bin/sh
set -eu

PASV_ADDRESS="${PASV_ADDRESS:-127.0.0.1}"
USERS_TXT="${USERS_TXT:-/var/run/users/users.txt}"
SHARED_DIR="${SHARED_DIR:-/shared}"
FTP_ROOT="${FTP_ROOT:-/srv/ftp}"

mkdir -p "$SHARED_DIR" "$FTP_ROOT"

# vsftpd.conf 렌더링
sed "s|__PASV_ADDRESS__|${PASV_ADDRESS}|g" \
    /etc/vsftpd/vsftpd.conf.template \
    > /etc/vsftpd/vsftpd.conf

# PAM 설정 배치
cp /etc/vsftpd/pam_vsftpd_virtual /etc/pam.d/vsftpd_virtual

# 초기 users.db 생성
if [ -f "$USERS_TXT" ]; then
    db_load -T -t hash -f "$USERS_TXT" "$SHARED_DIR/users.db"
    chmod 600 "$SHARED_DIR/users.db"

    # 사용자 디렉토리 보장
    awk 'NR%2==1' "$USERS_TXT" | while read -r u; do
        mkdir -p "$FTP_ROOT/$u"
        chown ftpvirt:ftpvirt "$FTP_ROOT/$u"
    done
else
    echo "WARN: $USERS_TXT not found, vsftpd will start but no users can log in" >&2
fi

# vsftpd 로그를 stdout으로 노출: named pipe + tail -F
# (vsftpd는 privsep 후 비특권 child에서 log_file을 open하므로 /dev/stdout 직접 사용 불가)
LOG_PIPE=/var/log/vsftpd.log
if [ ! -p "$LOG_PIPE" ]; then
    rm -f "$LOG_PIPE"
    mkfifo "$LOG_PIPE"
    chmod 666 "$LOG_PIPE"
fi
tail -F "$LOG_PIPE" &

# vsftpd 기동 (foreground)
exec /usr/sbin/vsftpd /etc/vsftpd/vsftpd.conf
```

`USERS_TXT` 형식은 `db_load -T -t hash` 입력 형식: 사용자명/패스워드를 줄바꿈으로 교차.

**로그 stdout 노출:** vsftpd는 privsep 후 비특권 child에서 `vsftpd_log_file`을 open하므로 `/dev/stdout` 직접 지정 시 `500 OOPS`로 모든 세션이 끊긴다. 따라서 conf는 일반 파일 경로(`/var/log/vsftpd.log`)를 지정하고, entrypoint에서 named pipe(FIFO)로 만들어 `tail -F`로 PID 1 stdout으로 흘려보낸다. kubectl logs로 FTP 프로토콜 라인 그대로 확인 가능.

### Task 5: user-syncer.sh 사이드카 스크립트 작성

**Files:**
- Create: `docker/user-syncer.sh`

- [ ] **Step 1: user-syncer 작성**

Write `docker/user-syncer.sh`:
```bash
#!/bin/sh
set -eu

USERS_TXT="${USERS_TXT:-/var/run/users/users.txt}"
SHARED_DIR="${SHARED_DIR:-/shared}"
FTP_ROOT="${FTP_ROOT:-/srv/ftp}"

regenerate() {
    if [ ! -f "$USERS_TXT" ]; then
        echo "WARN: $USERS_TXT 없음, 동기화 스킵" >&2
        return 0
    fi

    # 형식 검증: 줄 수가 짝수여야 함 (user/password 쌍)
    lines=$(wc -l < "$USERS_TXT")
    if [ $((lines % 2)) -ne 0 ]; then
        echo "ERROR: $USERS_TXT 줄 수가 짝수가 아님 (현재 $lines), 기존 DB 유지" >&2
        return 1
    fi

    # 사용자명 유효성 (영문/숫자/언더스코어/하이픈만)
    awk 'NR%2==1' "$USERS_TXT" | while read -r u; do
        case "$u" in
            *[!a-zA-Z0-9_-]*|"")
                echo "ERROR: 잘못된 사용자명 '$u', 기존 DB 유지" >&2
                exit 1
                ;;
        esac
    done || return 1

    db_load -T -t hash -f "$USERS_TXT" "$SHARED_DIR/users.db.new"
    chmod 600 "$SHARED_DIR/users.db.new"
    mv "$SHARED_DIR/users.db.new" "$SHARED_DIR/users.db"

    awk 'NR%2==1' "$USERS_TXT" | while read -r u; do
        if [ ! -d "$FTP_ROOT/$u" ]; then
            mkdir -p "$FTP_ROOT/$u"
            chown ftpvirt:ftpvirt "$FTP_ROOT/$u"
            echo "INFO: 사용자 디렉토리 생성: $u"
        fi
    done

    echo "INFO: users.db 동기화 완료 ($(date -Iseconds))"
}

# 초기 동기화
regenerate || true

# inotify 감시 루프
inotifywait -m -e modify -e create -e delete_self -e moved_to "$(dirname "$USERS_TXT")" |
while read -r _ event _; do
    echo "INFO: 변경 감지 ($event), 동기화 시작"
    sleep 1   # k8s가 마운트를 갱신하는 동안 잠시 대기
    regenerate || echo "ERROR: 동기화 실패, 다음 이벤트 대기"
done
```

```bash
set -eu

CONTAINER_NAME="k8s-ftp-smoke"
USERS_FILE=$(mktemp)
TESTFILE=$(mktemp)
DOWNLOADED=$(mktemp)

bob

set ftp:passive-mode true
set xfer:clobber on
put $TESTFILE -o test.txt
get test.txt -o $DOWNLOADED
bye
LFTP

# 업로드 검증: 컨테이너 내부에 파일이 실제로 존재
docker exec "$CONTAINER_NAME" test -f /srv/ftp/alice/test.txt
echo PASS_UPLOAD

# 다운로드 내용 검증
if diff -q "$TESTFILE" "$DOWNLOADED" >/dev/null 2>&1; then
    echo PASS_DOWNLOAD
else
    echo FAIL_DOWNLOAD >&2
    exit 1
fi

echo "OK: PASV 스모크 테스트 통과"
```

```bash
set -eu

CONTAINER_NAME="k8s-ftp-chroot"
USERS_FILE=$(mktemp)

cat > "$USERS_FILE" <<'EOF'
alice
alicepw
bob
bobpw
EOF

cleanup() {
    docker rm -f "$CONTAINER_NAME" >/dev/null 2>&1 || true
    rm -f "$USERS_FILE"

    -p 2121:21 \

    -e PASV_ADDRESS=127.0.0.1 \

    --cap-add NET_BIND_SERVICE \
    --cap-add SYS_CHROOT \
    k8s-ftp:dev

sleep 3

# bob이 자기 디렉토리에 파일 생성
docker exec "$CONTAINER_NAME" sh -c "echo 'bob-secret' > /srv/ftp/bob/secret.txt && chown ftpvirt:ftpvirt /srv/ftp/bob/secret.txt"

# alice로 로그인 후 bob 디렉토리 접근 시도
OUTPUT=$(lftp -u alice,alicepw -p 2121 -e "
    set ftp:passive-mode true;
    cd /bob 2>&1;
    cd ../bob 2>&1;
    cd ../../bob 2>&1;
    cd ../../../bob 2>&1;
    pwd;
    bye;
" 127.0.0.1 2>&1 || true)

# alice가 /bob에 진입하지 못해야 함 (자기 chroot 내에 bob 디렉토리 없음)
if echo "$OUTPUT" | grep -q "bob-secret"; then
    echo "FAIL: alice가 bob 데이터에 접근함"
    exit 1
fi

# 현재 디렉토리가 / 또는 chroot 루트여야 함 (escape 불가)
if echo "$OUTPUT" | grep -qE "pwd.*/bob"; then
    echo "FAIL: alice가 bob 디렉토리로 이동함"
    exit 1
fi

echo "OK: chroot 격리 확인"
```

```bash
#!/bin/sh
set -eu

CONTAINER_NAME="k8s-ftp-multi"
USERS_FILE=$(mktemp)

# 10명 사용자 생성
> "$USERS_FILE"
for i in $(seq 1 10); do
    printf "user%d\npass%d\n" "$i" "$i" >> "$USERS_FILE"
done

cleanup() {
    docker rm -f "$CONTAINER_NAME" >/dev/null 2>&1 || true
    rm -f "$USERS_FILE" /tmp/multiuser-*.txt
}
trap cleanup EXIT

docker run -d --name "$CONTAINER_NAME" \
    -p 2121:21 \
    -p 30000-30099:30000-30099 \
    -e PASV_ADDRESS=127.0.0.1 \
    -v "$USERS_FILE:/var/run/users/users.txt:ro" \
    --cap-add NET_BIND_SERVICE \
    --cap-add SYS_CHROOT \
    k8s-ftp:dev

sleep 3

# 10명 동시 업로드
pids=""
for i in $(seq 1 10); do
    TESTFILE="/tmp/multiuser-$i.txt"
    echo "data from user$i" > "$TESTFILE"
    lftp -u "user$i,pass$i" -p 2121 -e "
        set ftp:passive-mode true;
        put $TESTFILE -o file.txt;
        bye;
    " 127.0.0.1 >/dev/null 2>&1 &
    pids="$pids $!"
done

# 모두 완료 대기
for pid in $pids; do
    wait "$pid"
done

# 각 사용자가 자기 디렉토리에 자기 파일만 있는지 확인
for i in $(seq 1 10); do
    EXPECTED="data from user$i"
    ACTUAL=$(docker exec "$CONTAINER_NAME" cat "/srv/ftp/user$i/file.txt")
    if [ "$ACTUAL" != "$EXPECTED" ]; then
        echo "FAIL: user$i 파일 불일치 (expected=$EXPECTED, actual=$ACTUAL)"
        exit 1
    fi
done

echo "OK: 10명 동시 업로드 격리 확인"
```

**Files:**

- [ ] **Step 1: 테스트 작성**

Write `tests/zero-downtime-useradd.sh`:
```bash
#!/bin/sh
# 사용법: ./zero-downtime-useradd.sh <LB_IP> <기존 alice 패스워드>
set -eu

LB_IP="${1:?사용법: $0 <LB_IP> <alice_pw>}"
ALICE_PW="${2:?사용법: $0 <LB_IP> <alice_pw>}"

# 1. 백그라운드에서 alice의 long-running 전송 시작
LARGE_FILE=$(mktemp)
dd if=/dev/urandom of="$LARGE_FILE" bs=1M count=200 2>/dev/null

lftp -u "alice,$ALICE_PW" "$LB_IP" -e "
    set ftp:passive-mode true;
    set net:limit-rate 5M:5M;
    put $LARGE_FILE -o longfile.bin;
    bye;
" &
LFTP_PID=$!

# 2. 10초 대기 (전송 진행 중)
sleep 10
echo "alice의 전송 진행 중..."

# 3. 신규 사용자 charlie 추가
START_TIME=$(date +%s)
kubectl get secret -n ftp vsftpd-users -o yaml | \
    sed '/users.txt:/,/^[^ ]/ s|^    bob$|    bob\n    bobpw\n    charlie\n    charliepw|' | \
    kubectl apply -f -

# 위 sed가 정확하지 않을 수 있으므로 안전한 대안: 별도 파일에서 patch
# kubectl create secret generic vsftpd-users --from-file=users.txt=new-users.txt --dry-run=client -o yaml | kubectl apply -f -

# 4. charlie로 로그인 시도, 성공할 때까지 폴링 (최대 120초)
for i in $(seq 1 24); do
    if lftp -u charlie,charliepw "$LB_IP" -e "set ftp:passive-mode true; cls; bye;" >/dev/null 2>&1; then
        END_TIME=$(date +%s)
        ELAPSED=$((END_TIME - START_TIME))
        echo "OK: charlie 로그인 성공 (소요 시간: ${ELAPSED}초)"
        break
    fi
    sleep 5
    if [ "$i" = 24 ]; then
        echo "FAIL: 120초 내에 charlie 로그인 실패"
        kill "$LFTP_PID" 2>/dev/null || true
        rm -f "$LARGE_FILE"
        exit 1
    fi
done

# 5. alice의 백그라운드 전송이 영향받지 않았는지 확인
wait "$LFTP_PID"
if [ $? -ne 0 ]; then
    echo "FAIL: alice의 진행 중 전송이 실패함"
    rm -f "$LARGE_FILE"
    exit 1
fi

echo "OK: alice 전송 완료, charlie 무중단 추가 검증 성공"
rm -f "$LARGE_FILE"
```
